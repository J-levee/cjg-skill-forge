# Feedback Loop · 不打扰用户的持续体验收集与迭代机制

**When to use**: Discipline 9。任何技能发布后都不是"造完就完"——它必须**边被使用、边被真实体验修正**。本文件解决一个核心矛盾：**既要持续拿到用户使用体验来迭代，又绝不能用弹窗问卷去打扰用户。**

> 第一原则：**默认只观察，几乎不发问。** 最好的反馈是你本来就在进行的对话里"顺手读出来"的行为信号，不是让用户额外填表。真正牛逼的技能，用户感觉不到它在收集反馈。

---

## 三层机制（打扰程度递增，频率递减）

### Tier 0 — 被动行为信号（永远开启 · 对用户零成本）
不问任何问题，只从**自然对话本身**推断满意度。每次技能被调用后，agent 读这些信号并静默记一行日志：

| 信号 | 用户表现（自然发生，非诱导） | 推断 |
|------|------------------------------|------|
| ✅ 采纳 | "好/明白/这个思路对"、按建议往下走、开始执行 | 命中，有效 |
| 🎯 深挖 | 追问更深一层的问题 | 高度投入，技能起作用 |
| ⚠️ 纠正 | "不对/不是这个意思/你理解错了" | 技能**误诊**——最有价值的信号 |
| 🔁 重问 | 换个说法把同一问题又问一遍 | 答案没落地 |
| 🛑 放弃 | 突然转话题 / 丢下不回 | 疑似不满意 |

**记录方式**：滚动日志一行/次，只存**模式标签**不存原文——如
`2026-07-11 | 触发类:开题 | 命中:three-gates,triage | 自评置信:高 | 用户信号:采纳`
放在技能可写的工作区日志（不塞进 SKILL.md 本体，避免膨胀）。

> **🛑 放弃 → 关④ 深度归因**：单个"放弃"信号只是 Tier 0 的一条噪声；但当某技能**持续**出现"放弃/转话题"且冷却期到（≥14 天未复用、曾调用≥2 次、曾达终态），按 `references/churn-reflector.md` 做**7 类根因归因**（E 自然完成前置过滤 / C1 找不到时机 / C2 卡点 / C3 输出不符 / C4 信任断裂 / C5 被替代 / C6 自身回归），产出带证据链的假设——这是关③（进迭代循环）的**对称补半环**，闭合增长飞轮出口。关④ 默认开启（与关② 萃取器默认关闭相反），零 PII、可关。

### Tier 1 — Agent 自评校准（永远开启 · 对用户零成本）
答完后 agent 诚实记自己这一手的成色（即 persona 技能里 `acquaintance.md` 的 calibration 段，此处推广到所有技能）：
- 实际调动了哪些资源？有没有**空转**（某资源从不被触发 = 死重，该砍）。
- 自评置信度是高/中/低？有没有**被迫猜**、撞到知识边界？
- 用户后续是否纠正了我 → 回填"老衲被纠正"闭环。

### Tier 2 — 极稀疏的 opt-in 微信号（罕用 · 严格限流）
仅当**同时满足**：①任务自然收尾的时刻 ②距上次询问已隔 N 次会话 ③用户没表达过"别打扰"。才可发**一个**轻量可选信号：
> "这次帮到你了吗？一句话就行，不想说忽略即可。"

铁律：
- **绝不**多问题问卷、**绝不**每条消息后要👍👎、**绝不**在任务中途打断。
- 用户一旦说"别问了/烦" → **永久关闭** Tier 2，只留 Tier 0/1。
- 对明确"讨厌被打扰"的用户，Tier 2 默认 OFF。

## 蒸馏 → 迭代闭环（信号变不成版本就白收集）
按节奏（每 N 次使用，或日志超过阈值）把信号蒸馏成**迭代 backlog**：

| 日志里反复出现的 | 说明 | 喂给哪条纪律 |
|------------------|------|--------------|
| 某类问题反复"纠正/重问" | 该类误诊/接不住 | → Forge 循环下一版 targeted fix |
| 某 reference 从不被触发 | 死重 | → 砍掉，提升 token 效率 (D3) |
| 反复撞知识边界 | 覆盖缺口 | → coverage-audit 补真材料 (D1) |
| 反复假笃定被纠正 | 置信度校准失灵 | → 收紧 confidence 规则 (D5) |

闭环全景：
```
上线前: 拟真测试(simulation-testing) ──┐
                                        ├─→ 发现缺口 → Forge 下一版 → 校验打包
上线后: 被动信号+自评(feedback-loop) ──┘         ↑                        │
              ↑                                   └────── 蒸馏 backlog ◄────┘
              └──────────────── 持续使用 ──────────────────────────────────┘
```
**一次达到 Global-Best 不算数;能靠这个环持续保持才算。** 把这条写进 Core principle。

## 隐私与安全红线
- 只记**方法论模式标签**（触发类/命中资源/信号类型），**绝不**记用户问题原文、身份、敏感内容。
- 日志**本地、用户所有、可随时删**;不外传、不进技能分发包。
- Tier 2 完全 opt-in;沉默即"不打扰"，不是"默认同意"。

## 反面模式（别踩）
- ❌ 弹窗问卷 / NPS 打分器——最粗暴的打扰，第一原则的反例。
- ❌ 每条消息后要用户评价——反馈疲劳，信噪比暴跌。
- ❌ 给说过"别打扰"的用户继续发 Tier 2——违背信任，一次即失。
- ❌ 存用户原文/PII——隐私红线。
- ❌ 靠**单条**反馈就改技能——需成簇/过阈值才动，单个轶事会带偏。
- ❌ 只收集不蒸馏——日志堆着不回流版本 = 白收集。
